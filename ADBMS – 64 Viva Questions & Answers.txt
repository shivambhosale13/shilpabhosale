Q1. What is data partitioning?
A: Dividing a large table into smaller parts for faster access.
Q2. Why do we use data partitioning?
A: To improve performance and manage large data.
Q3. What is RANGE partitioning?
A: Partitioning based on a range of values like years.
Q4. Which keyword is used in RANGE partitioning?
A: PARTITION BY RANGE.
Q5. Can ranges overlap in RANGE partitioning?
A: No, ranges should not overlap.
Q6. What is LIST partitioning?
A: Partitioning based on fixed or discrete values.
Q7. Which keyword is used in LIST partitioning?
A: VALUES IN.
Q8. Which partitioning is better for year-wise data?
A: RANGE partitioning.
Q9. Which partitioning is used for fixed values?
A: LIST partitioning.
Q10. What happens if partition is dropped?
A: Data in that partition is deleted.
Q11. What is OLAP?
A: Online Analytical Processing used for analysis.
Q12. What is ROLLUP?
A: Generates subtotals and grand total.
Q13. What is CUBE?
A: Generates all possible subtotal combinations.
Q14. ROLLUP vs CUBE?
A: CUBE gives more combinations than ROLLUP.
Q15. What is an analytic function?
A: Works on group of rows and returns value for each row.
Q16. What is RANK?
A: Assigns rank with gaps.
Q17. What is DENSE_RANK?
A: Assigns rank without gaps.
Q18. Difference between RANK and DENSE_RANK?
A: DENSE_RANK does not skip numbers.
Q19. What is LAG?
A: Fetches previous row value.
Q20. What is LEAD?
A: Fetches next row value.
Q21. Which function compares previous row?
A: LAG.
Q22. Which function compares next row?
A: LEAD.
